These are the commands you'll use 90% of the time.

Command Description Example
git init Initialize a new Git repo in the current folder. git init
git clone <url> Download a remote repository to your machine. git clone https://github.com/user/repo.git
git status Your best friend. Shows the state of your working directory and staging area. git status
git add <file> Stage a specific file for the next commit. git add index.html
git add . Stage all new and modified files for the next commit. git add .
git commit -m "msg" Take a snapshot of the staged changes with a descriptive message. git commit -m "Add login button"
git push origin <branch> Upload your local commits to a remote repository (like GitHub). git push origin main
git pull origin <branch> Download changes from the remote and merge them into your current branch. git pull origin main

Working on multiple features without messing up the main code.

Command Description Example
git branch List all local branches. The * shows your current branch. git branch
git branch <name> Create a new branch based on your current commit. git branch new-feature
git checkout <branch> Switch to an existing branch. git checkout new-feature
git checkout -b <name> Create a new branch and switch to it immediately. (Very common) git checkout -b fix-bug
git switch <branch> Newer, clearer way to switch branches (Git 2.23+). git switch main
git switch -c <name> Newer way to create and switch to a branch. git switch -c new-feature
git merge <branch> Combine the specified branch's history into your current branch. git switch main
git merge new-feature
git branch -d <branch> Delete a branch that has been merged (safe delete). git branch -d old-feature
git branch -D <branch> Force delete a branch, even if it hasn't been merged. (Be careful!) git branch -D abandoned-experiment

The "oh crap" moments. Use these carefully.

Command Description When to Use Example
git restore <file> Discard changes in your working directory for a specific file. "I haven't added this file yet and I want to start over." git restore README.md
git restore --staged <file> Unstage a file without losing the changes in your working directory. "I added the wrong file by accident." git restore --staged README.md
git commit --amend Edit the most recent commit. Change the message or add forgotten files. "I just committed but forgot a file!" or "My commit message has a typo." git commit --amend -m "A better commit message"
git reset --soft HEAD~1 Undo the last commit but keep all your changes as staged. "I want to combine my last two commits." git reset --soft HEAD~1
git reset --hard HEAD~1 Completely obliterate the last commit and all its changes. Warning: Unrecoverable. "That last commit was a huge mistake, I want to pretend it never happened." git reset --hard HEAD~1
git revert <commit-hash> Create a new commit that undoes the changes of a previous commit. Safe for public history. "I need to undo a commit I already pushed to others." git revert a1b2c3d4

For a clean, professional commit history.

Command Description Use Case Example
git rebase <branch> Replay your current branch's commits on top of the latest version of <branch>. "I want a clean, linear history for my feature branch before merging." git rebase main
git rebase -i HEAD~3 Interactive Rebase. Opens an editor to squash, edit, reorder, or drop commits. "I have 5 messy 'WIP' commits I want to combine into one clean commit." git rebase -i HEAD~3
git cherry-pick <commit> Apply the changes from a specific commit to your current branch. "I need just that one bugfix from another branch, not the whole thing." git cherry-pick a1b2c3d4
git stash Temporarily shelves (stashes) all your uncommitted changes. "My boss needs a fix NOW, but I'm in the middle of something." git stash
git stash pop Re-apply your most recently stashed changes and remove them from the stash. "Okay, boss's fix is done, back to my feature." git stash pop

Figuring out what happened and when.

Command Description Example
git log Show the commit history. git log
git log --oneline --graph --all Fancy, compact history view with branch topology. Memorize this. git log --oneline --graph --all
git diff Show unstaged changes since the last commit. git diff
git diff --staged Show staged changes (what you've added). git diff --staged
git blame <file> Show who last modified each line of a file and in which commit. git blame README.md

Working with others on GitHub/GitLab.

Command Description Example
git fetch origin Download all changes from the remote without merging them. "See what others have done." git fetch origin
git pull --rebase origin main Best practice pull. Fetches remote changes and rebases your commits on top of them. Cleaner than a merge. git pull --rebase origin main

Deleting a Remote Branch in Git

Deleting a remote branch is a common task when you've merged a feature branch and want to clean up. Here's how to do it:

The Command

git push origin --delete branch-name

or the shorter equivalent:

git push origin :branch-name

Step-by-Step Explanation

  1. First, check what remote branches exist: 
    git branch -r
  2. Delete the remote branch: 
    # Example: delete a branch named "feature-login"
git push origin --delete feature-login
  3. Verify it's gone: 
    git branch -r  # Should no longer show the deleted branch

Important Notes

  • You can't delete the default branch (usually main or master) unless you change the default first
  • Other team members will need to run git fetch --prune to sync their local remote-tracking branches
  • Deleting a remote branch doesn't delete local branches - you'll need to do that separately

Example Workflow

# List all remote branches
git branch -r

# Delete the remote feature-login branch
git push origin --delete feature-login

# Also delete the local branch if you're done with it
git branch -d feature-login

# Clean up remote-tracking references
git fetch --prune

What If You Get an Error?

If you see an error like remote ref does not exist, it means:

  1. The branch was already deleted, or
  2. You typed the branch name incorrectly

Always double-check with git branch -r first!

If you are having trouble pushing to a remote repository due to a "Permission denied" error, you might have an issue with your SSH key configuration. Here is a step-by-step guide to check and fix it.

Step 1: Check for Existing SSH Keys

First, check if you have an SSH key file. The most common filenames are id_rsa, id_ed25519, or id_ecdsa. You can list the files in your ~/.ssh directory with this command:

ls -al ~/.ssh

If you see files like id_ed25519 and id_ed25519.pub, you have an SSH key.

Step 2: Start the SSH Agent

The ssh-agent is a background program that handles your SSH keys. You need to make sure it's running. You can start it with this command in your terminal:

eval "$(ssh-agent -s)"

You should see a message like Agent pid 12345.

Step 3: Add Your SSH Key to the Agent

Once the agent is running, you need to add your SSH key to it. If your key file is id_ed25519, you can use this command:

ssh-add ~/.ssh/id_ed25519

If it asks for a passphrase, enter the one you set for your SSH key. You should see a confirmation like Identity added.

Step 4: Test Your Connection to GitHub

Finally, test your connection to GitHub with this command:

ssh -T git@github.com

If everything is configured correctly, you should see a message like this:

Hi username! You've successfully authenticated, but GitHub does not provide shell access.

If you see this message, your SSH key is correctly configured, and you should be able to push to your repository.

This guide will walk you through the process of setting up Git and GitHub on a new computer, from installation to connecting to your GitHub account.

Step 1: Install Git

First, you need to install Git on your computer. You can download it from the official website: https://git-scm.com/downloads. Follow the installation instructions for your operating system.

Step 2: Configure Your Git Identity

Once Git is installed, you need to configure your name and email address. This information will be used for every commit you create.

git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

Replace "Your Name" and "your.email@example.com" with your actual name and the email address you will use for your GitHub account.

Step 3: Create a New GitHub Account

If you don't have a GitHub account, go to https://github.com/join and create a new account. Follow the instructions on the website.

Step 4: Generate a New SSH Key

An SSH key is used to securely connect your computer to your GitHub account. To generate a new SSH key, open your terminal and run the following command:

ssh-keygen -t ed25519 -C "your.email@example.com"

When you're prompted to "Enter a file in which to save the key," you can press Enter to accept the default file location. You will also be asked to enter a passphrase, which is highly recommended for security.

Step 5: Add Your SSH Key to Your GitHub Account

Now you need to add your new SSH key to your GitHub account. First, you need to copy the public key to your clipboard. You can do this with the following command:

pbcopy < ~/.ssh/id_ed25519.pub

If `pbcopy` is not available on your system, you can open the id_ed25519.pub file with a text editor and copy the content manually.

Next, go to your GitHub account settings, click on "SSH and GPG keys", and then click on "New SSH key". Give your key a title (e.g., "My New Laptop") and paste the copied key into the "Key" field. Finally, click "Add SSH key".

Step 6: Test Your Connection

To make sure everything is working, you can test your connection to GitHub with this command:

ssh -T git@github.com

You should see a message like "Hi username! You've successfully authenticated...".

Congratulations! You have successfully set up Git and GitHub on your new computer.
