Initial Configuration

Configure your Git identity and settings. This is a one-time setup.

# Set your name for all repositories
git config --global user.name "Your Name"

# Set your email for all repositories
git config --global user.email "you@example.com"

# View all global config settings
git config --list

# Get a specific config value
git config --get user.name
üí° Pro Tip
The --global flag applies these settings to every repo on your system. Omit it to set config for the current repo only.

Feature Branch Workflow

Standard workflow for feature development in teams.

# Create and switch to feature branch
git checkout -b feature/user-authentication

# Make your changes, then commit
git add .
git commit -m "Add user authentication system"

# Push feature branch
git push -u origin feature/user-authentication

# Switch back to main and merge
git checkout main
git merge feature/user-authentication
üí° Pro Tip
After merging, you can delete the branch. See the "Deleting & Destroying" tab for how to do this safely.

Stashing Work

Temporarily save uncommitted changes.

# Save current changes
git stash

# Save with a message
git stash push -m "WIP: refactoring auth logic"

# List stashes
git stash list

# Apply latest stash
git stash pop

# Apply specific stash
git stash pop stash@{1}

# Apply without removing from stash
git stash apply

# Delete stash
git stash drop stash@{1}

Wrong Git Root

Problem:
You ran git init in the wrong directory and Git is tracking too many files.
Solution:
# Navigate to wrong directory
cd ~/Documents

# Check if Git repo exists
ls -a

# Remove the wrong Git repo
rm -rf .git

# Navigate to correct project folder
cd ~/Documents/my-project

# Initialize Git in correct location
git init
git add .
git commit -m "Initial commit"

Wrong Remote URL

# Push to correct repository

Undoing Commits

Various ways to undo changes depending on your situation.

# Unstage file (keep changes)
git reset HEAD filename

# Undo last commit, keep changes staged
git reset --soft HEAD~1

# Undo last commit, keep changes unstaged
git reset HEAD~1

# Create new commit that undoes previous commit
git revert abc1234
‚ö†Ô∏è Warning
To permanently discard changes, see the "Discard Local Changes" card in the "Deleting & Destroying" tab.

Interactive Rebase

Rewrite commit history for cleaner project timeline.

# Rebase last 3 commits interactively
git rebase -i HEAD~3

# Common rebase commands in editor:
# pick = use commit
# reword = edit commit message
# edit = stop to modify commit
# squash = combine with previous commit
# fixup = like squash but discard message
# drop = remove commit

# Force push after rebase (be careful!)
git push --force-with-lease
‚ö†Ô∏è Warning
Never rebase commits that have been pushed to shared branches. Only rebase private feature branches.

Performance Optimization

Optimize Git performance for large repositories.

# Enable parallel processing
git config core.preloadindex true
git config core.fscache true

# Shallow clone for faster downloads
git clone --depth 1 https://github.com/user/repo.git

# Partial clone (Git 2.19+)
git clone --filter=blob:none https://github.com/user/repo.git

# Enable Git LFS for large files
git lfs install
git lfs track "*.zip"
git add .gitattributes

# Configure Git for better performance
git config pack.threads 4
git config pack.windowMemory 100m

Delete Branches

Remove local and remote branches.

# Delete a local branch (safe)
# Won't delete if it has unmerged changes
git branch -d feature/old-feature

# Delete a local branch (force)
# Deletes branch even with unmerged changes
git branch -D feature/old-feature

# Delete a remote branch
git push origin --delete feature/old-feature
üí° Pro Tip
Clean up merged branches regularly to keep your repository tidy.

Remove Files & Untrack

Remove files from Git's tracking index.

# Stop tracking a file but keep it locally
# (e.g., a config file that was accidentally committed)
git rm --cached config.js

# Remove a file from both Git and filesystem
git rm file.log

# Clean untracked files from working directory
# -n: dry run (shows what would be deleted)
git clean -n
# -f: force (actually deletes the files)
git clean -f
# -fd: force delete files and directories
git clean -fd

Discard Local Changes

Permanently throw away uncommitted changes.

DANGER ZONE:
These commands permanently destroy work. There is no undo. 
# Discard changes in a specific file
git checkout -- path/to/file.js

# Discard all local changes in all files
git reset --hard

# Discard all local changes and remove all untracked files
git reset --hard && git clean -fd

Remove from History

Completely remove a file from the entire project history.

NUCLEAR OPTION:
This rewrites history. All collaborators will need to re-clone or perform a complex rebase. Use with extreme caution. 
# Use the modern git-filter-repo tool
# (Install first: pip install git-filter-repo)

# Completely remove a file from all commits
git filter-repo --path path/to/sensitive-file.txt --invert-paths

# Remove a folder from all commits
git filter-repo --path path/to/folder/ --invert-paths --force

# Push the rewritten history
# You MUST coordinate with your team before this step
git push --force-with-lease
